Giới Thiệu Bản Thân

Sinh viên ngành An toàn thông tin định hướng chuyên sâu vào Giám sát an ninh mạng và Phản ứng sự cố (SOC Tier 1). Sở hữu tư duy "Offensive-driven Defense" (Dùng kiến thức tấn công để tối ưu hóa phòng thủ) giúp nhận diện nhanh chóng các hành vi bất thường, giải mã payload độc hại và cấu hình luật phát hiện trên hệ thống SIEM.

Kiến Thức & Kỹ Năng

Security Monitoring & OSINT

  • Phân tích log: Windows Event Logs (Security, Sysmon), Linux Syslogs.
  • Điều tra IoC qua Threat Intel: VirusTotal, AbuseIPDB, Shodan.
  • Sử dụng nền tảng: Mô phỏng SOC (LetsDefend), ý niệm SIEM (Splunk, Wazuh).

Vulnerability & Vectors

  • Hiểu sâu lỗ hổng OWASP Top 10 (SQLi, XSS, RCE, File Upload).
  • Sử dụng công cụ rà quét/đọc vết mạng: Burp Suite, Nmap, Wireshark, SQLMap.
  • Tư duy ánh xạ chuỗi hành vi xâm nhập vào khung MITRE ATT&CK.

Hạ Tầng Mạng & HĐH

  • Kiến thức TCP/IP, DNS, Routing/Switching, DMZ, Network Segmentation.
  • Quản trị và thao tác tốt trên hệ điều hành: Linux (Ubuntu/Kali), Windows Server.
  • Kinh nghiệm môi trường ảo hóa: Docker, WSL2, GNS3.

Lập Trình & Tự Động Hóa

  • Viết mã script phân tích/bóc tách dữ liệu log thô bằng Python và Bash.
  • Hiểu cơ chế hoạt động, cơ chế ghi log của mã nguồn backend (Java Spring Boot).

SOC Investigation Write-ups

Nơi lưu trữ và chia sẻ quá trình phân tích, điều tra các sự cố bảo mật thực tế từ các nền tảng Lab (LetsDefend, CyberDefenders, v.v.).

Ngày hoàn thành: 01/06/2026  |  Nguồn: LetsDefend Alert Log
SQL Injection Log Analysis SQLMap HTTP 200 OK

Phân tích tệp nhật ký truy cập (Access Log) từ máy chủ Web chịu sự cố quét lỗ hổng hàng loạt từ IP nguồn thuộc khu vực Indonesia. Tiến hành giải mã payload hỗn hợp (UNION + xp_cmdshell) và đánh giá độ chính xác mã trạng thái phản hồi.

[Sysmon Investigation] Phát hiện chuỗi hành vi mã độc thực thi qua file Word chứa Macro (.DOCM)
Ngày hoàn thành: 28/05/2026  |  Nguồn: Windows Sysmon Event
Phishing Malware Macro Sysmon Event ID 1 PowerShell Bypass

Theo dõi dấu vết tiến trình từ sự kiện Sysmon Event ID 1 (Process Create). Phân tích cách tệp Word độc hại kích hoạt script PowerShell ngầm để thực hiện bypass chính sách thực thi hệ thống (Execution Policy Bypass) nhằm thiết lập ban đầu.
